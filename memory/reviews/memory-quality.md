# Memory Quality Score

> 评估 Captain 记忆系统质量的打分标准

---

## 评估维度

### 1. 完整性（Completeness）— 25%
- [ ] 所有重要领域都有记录？
- [ ] 缺失哪些领域？
- [ ] Sources 层数据是否定期同步？

### 2. 准确性（Accuracy）— 30%
- [ ] 信息还正确吗？
- [ ] 哪些过时了？
- [ ] MEMORY.md 的服务状态是否已更新？

### 3. 时效性（Freshness）— 25%
- [ ] 最后更新是何时？
- [ ] 超过30天未更新的标记⚠️
- [ ] 每日日志是否连续？

### 4. 可检索性（Retrievability）— 20%
- [ ] 能通过 index.md 快速找到？
- [ ] 搜索是否有效？
- [ ] 分类是否清晰？

---

## 评分标准

| 等级 | 分值 | 说明 |
|------|------|------|
| 优秀 | 9-10 | 四维全绿，无需改进 |
| 良好 | 7-8 | 轻微问题，不影响使用 |
| 及格 | 5-6 | 部分过时/缺失，需要更新 |
| 危险 | 3-4 | 严重过时，需紧急修复 |
| 失效 | 1-2 | 记忆系统基本不可用 |

---

## 各层质量要求

### Layer 1: MEMORY.md
- 铁律、人物、项目、教训必须准确
- 服务状态需标注日期
- 超过30天未更新自动⚠️

### Layer 2: Daily Logs
- 每日一条，记录重要事件
- 连续性检查（有无缺失日期）

### Layer 3: Sources + Patterns
- Sources 保留原始数据来源
- Patterns 提炼模式，驱动行为优化

### Layer 4: Reviews
- 触发后生成评估报告
- 输出 Action Items
- 下次自审复查

---

## 自审触发条件

满足任一即触发自审：
1. 每周一自动自审
2. MEMORY.md 超过30天未更新
3. 发现新错误模式时
4. 用户指出记忆错误时
5. 重大系统变更后（如 OpenClaw 升级）
