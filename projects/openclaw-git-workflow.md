# OpenClaw Git 工作流增强设计

## 一、现状分析

### 1.1 现有 Skill

| Skill | 实现方式 | 定位 |
|-------|---------|------|
| `github` | `gh` CLI 封装 | 基础：PR/issue/CI 查询，手动操作 |
| `gh-issues` | curl + REST API + sub-agent | 高级：自动抓 issue + 修复 + PR review |

**GitHub skill 当前能力：**
- `gh pr list / view / create / merge`
- `gh issue list / create / close`
- `gh run list / view / rerun`
- `gh api` 原始查询

**gh-issues skill 工作流（参考实现）：**
- 自动检测 dirty tree
- 自动创建 `fix/issue-{N}` 分支
- 自动 commit + push + PR 创建
- 自动处理 review comments

### 1.2 增强点

| 现有缺失 | 说明 |
|---------|------|
| `/commit` | 没有自动变更检测 + 智能 commit message |
| `/branch` | 没有分支创建/切换/管理命令 |
| `/commit-push-pr` | 没有三合一命令 |
| 分支策略 | 没有基于 issue 的分支命名规范 |
| commit message AI | 没有 conventional commit 生成 |
| 冲突检测 | 没有 push 前冲突检查 |
| PR template | 没有自动填充 PR body |

## 二、设计方案

### 2.1 命令体系

```
/commit [options]        — 智能 commit
/branch [options]       — 分支管理
/commit-push-pr [opts]   — 三合一工作流
/pr [options]            — PR 操作（增强现有）
/issue-branch <issue>     — 从 issue 创建分支
```

### 2.2 `/commit` 设计

**功能：**
1. 自动检测 staged / unstaged 变更
2. 生成符合 Conventional Commits 的 message
3. 支持 `--amend` 修正
4. 支持 `--dry-run` 预览

**实现：**

```bash
# 1. 检测变更
git status --porcelain
git diff --stat

# 2. 分类变更类型
# - 新文件 → feat
# - 修改文件 → feat / fix / refactor（根据路径判断）
# - 删除文件 → chore
# - 测试文件 → test

# 3. AI 生成 message（可选）
# 用模型分析 diff，生成符合 conventional commits 的 message
# Format: <type>(<scope>): <description>
# e.g. feat(clawflow): add cron trigger support

# 4. 执行 commit
git add -A
git commit -m "<generated message>"
```

**参数：**
- `--type=<type>` — 强制指定 type（feat/fix/chore/refactor/docs/test）
- `--scope=<scope>` — 指定 scope
- `--amend` — amend 上次 commit
- `--dry-run` — 只显示将要生成的 message，不执行

**输出示例：**
```
📦 变更检测：
  ✅ 新增   (3) src/handlers/cron.ts, src/db.ts
  ✏️  修改   (1) README.md
  ❌ 删除   (0)

📝 建议 Commit Message：
  feat(clawflow): add cron trigger support with interval config

[Enter] 确认提交 | [e] 编辑 message | [a] amend | [c] 取消
```

### 2.3 `/branch` 设计

**功能：**
1. 列出所有分支（含远程）
2. 创建新分支（支持从 issue 自动命名）
3. 切换分支
4. 删除分支（本地 + 远程）
5. 显示分支状态（vs origin/main）

**命令：**
```
/branch list              — 列出本地分支
/branch list --remote     — 列出远程分支
/branch create <name>     — 创建分支
/branch create fix/42     — 从 issue 创建 fix 分支
/branch switch <name>     — 切换分支
/branch delete <name>     — 删除本地分支
/branch delete <name> --remote — 删除远程分支
/branch status            — 当前分支状态
```

**实现：**
```bash
git branch [-a]                    # 列出分支
git checkout -b <branch> [base]   # 创建+切换
git push -u origin <branch>        # 推送并设置上游
git branch -d <branch>            # 安全删除
git push origin --delete <branch> # 远程删除
```

### 2.4 `/commit-push-pr` 设计（三合一核心命令）

**功能：** 一条龙执行 `commit → push → PR create`

**流程：**
```
1. 检测变更 + 生成 commit message
2. 确认 commit（可交互编辑）
3. push 到远程（自动创建分支如果不存在）
4. 自动创建 PR（从分支到默认分支）
5. 返回 PR URL
```

**参数：**
- `--message=<msg>` — 直接指定 commit message（跳过 AI 生成）
- `--title=<title>` — PR title（默认 = commit message 第一行）
- `--body=<body>` — PR body
- `--base=<branch>` — 目标分支（默认从仓库检测）
- `--dry-run` — 预览所有步骤，不执行
- `--yes` — 跳过所有确认

**实现步骤：**

```bash
# Step 1: 分支不存在则创建
git rev-parse --abbrev-ref HEAD  # 当前分支
git push -u origin HEAD         # 推送当前分支

# Step 2: 创建 PR（gh CLI 或 curl）
gh pr create \
  --title "$(git log -1 --format='%s')" \
  --body "## Summary\n\n<auto-generated from commits>" \
  --base main
```

**输出示例：**
```
✅ Committed: feat(search): add fuzzy matching
🚀 Pushed:    origin feature/search → new branch
🔗 PR Created: https://github.com/owner/repo/pull/123
```

### 2.5 `/issue-branch <issue>` 设计

**功能：** 从 GitHub issue 自动创建分支

**实现：**
```bash
# 获取 issue 信息
gh issue view <issue> --json number,title

# 生成分支名：fix/42-add-fuzzy-search
SLUG=$(echo "<title>" | slugify)
BRANCH="fix/<issue>-<SLUG>"

# 创建并切换
git checkout -b $BRANCH
```

### 2.6 PR 增强（对现有 `/pr` 命令）

**新增能力：**
- `/pr checks <pr>` — 查看 CI 状态 + 阻塞原因
- `/pr diff <pr>` — 显示 PR diff
- `/pr approve <pr>` — 批准 PR（需有权限）
- `/pr ready <pr>` — 标记 PR 为 ready for review
- `/pr draft <pr>` — 转为 draft

## 三、实现方案

### 3.1 新 Skill 文件结构

```
skills/
  git-workflow/
    SKILL.md          # 主技能文件，定义所有命令
    scripts/
      detect-changes.sh
      gen-commit-msg.sh
      create-pr.sh
```

### 3.2 核心脚本

#### `detect-changes.sh`
```bash
#!/bin/bash
# 检测变更并分类
git status --porcelain | while read line; do
  status="${line:0:2}"
  file="${line:3}"
  # 分类：新增/修改/删除
done
git diff --stat
```

#### `gen-commit-msg.sh`
```bash
#!/bin/bash
# 用 AI 生成符合 conventional commits 的 message
DIFF=$(git diff --cached)
# 调用模型分析 diff，输出 <type>(<scope>): <description>
```

#### `create-pr.sh`
```bash
#!/bin/bash
# 封装 gh pr create
gh pr create \
  --title "$TITLE" \
  --body "$BODY" \
  --base "${BASE:-main}" \
  --assignee "@me"
```

### 3.3 依赖

- `git` — 已有
- `gh` — 已有
- `node` / `bash` — 执行脚本
- 可选：模型调用生成 commit message

## 四、优先级

| 优先级 | 任务 | 理由 |
|-------|------|------|
| P0 | `/commit` | 最常用，收益最高 |
| P0 | `/branch` | 基础能力 |
| P0 | `/commit-push-pr` | 核心三合一 |
| P1 | `/issue-branch` | 与 gh-issues skill 协同 |
| P1 | PR 增强（checks/diff/approve） | review 效率 |
| P2 | AI 生成 commit message | 需要模型调用 |
| P2 | 冲突检测 + 提示 | push 前安全检查 |

## 五、与现有 gh-issues skill 的协同

`gh-issues` skill 已实现：
- ✅ 分支创建 `fix/issue-{N}`
- ✅ commit + push + PR 创建（通过 sub-agent）

`git-workflow` skill 将提供：
- ✅ 用户可直接触发的 `/commit` 等命令（不需要跑完整 issue 流程）
- ✅ 交互式 commit message 确认
- ✅ 更灵活的分支管理
- ✅ 三合一命令降低操作步骤

两者分工：
- `gh-issues` — 批量自动修复（机器为主）
- `git-workflow` — 日常开发工作流（人机交互）

## 六、风险与注意事项

1. **Dirty tree 处理**：push 前必须检查 uncommitted changes
2. **分支保护**：`main/master` 不能作为源分支
3. **Force push**：`--force` 需要用户显式确认
4. **Fork 模式**：参考 gh-issues skill 的 fork remote 处理
5. **gh auth**：确保 `gh auth status` 正常

## 七、附录：参考实现

gh-issues skill 中的分支/PR 创建逻辑（已验证可用）：

```bash
# 分支创建
git checkout -b fix/issue-{N} {BASE_BRANCH}

# Push with token auth
git remote set-url origin https://x-access-token:$GH_TOKEN@github.com/{REPO}.git
git push -u origin fix/issue-{N}

# PR 创建 via gh CLI
gh pr create --title "fix: {title}" --body "{body}" --base {BASE}
```
